# 模块功能清单

## 一、项目概述

**PARAM_TOOL** - 海关参数表全生命周期管理系统，基于Django框架开发，覆盖参数表从需求登记、元数据管理、AI辅助分析、代码自动生成到测试验证的完整流程。

## 三、核心模块说明

### 3.1 数据模型模块 (parameter/models.py)

**文件**: [models.py](file:///d:/CS/trae/PARAM_TOOL/parameter/models.py)

**职责**: 定义系统所有数据模型及其字段、关系、业务方法

| 模型类 | 功能说明 | 关键字段 |
|--------|----------|----------|
| `ParameterTable` | 参数表业务说明底账 | 名称、业务说明、所属领域、负责人、状态、版本 |
| `Metadata` | 元数据配置 | 字段类型、长度、控件类型、默认值、校验规则、描述 |
| `FieldDefinition` | 字段定义 | 字段名、显示名、类型、长度、存储方式、是否必填、排序号 |
| `Requirement` | 业务需求登记 | 需求编号、标题、类型、关联参数表、状态、故事点、冲刺 |
| `TaskDocument` | 任务书管理 | 关联需求、文档类型、内容、生成时间、导出时间 |
| `ConfigScript` | 配置脚本 | 关联需求、脚本类型、脚本内容、状态、目标环境 |
| `IndexIdConfig` | INDEXID配置 | 关联参数表、INDEXID、自定义列名、显示字段、校验规则 |
| `TestCase` | 测试用例 | 关联需求、用例编号、类型、前置条件、步骤、预期结果 |
| `AutomationTestResult` | 自动化测试结果 | 关联测试用例、执行状态、错误信息、执行时长 |
| `Role` | 角色 | 角色编码、角色名称、描述 |
| `User` | 用户 | 用户名、密码、真实姓名、邮箱、手机号、角色、状态 |

**存储方式设计**:
- `code_only`: 仅存储代码值（业务表引用参数表时使用）
- `code_name`: 存储代码和名称（参数表自身存储使用）
- `full`: 完整存储所有字段（需要保留完整信息的场景）

### 3.2 认证与用户管理模块 (parameter/auth_views.py)

**文件**: [auth_views.py](file:///d:/CS/trae/PARAM_TOOL/parameter/auth_views.py)

**职责**: 用户认证、权限控制、用户管理

#### 装饰器

| 装饰器 | 功能说明 |
|--------|----------|
| `login_required` | 登录验证，未登录重定向到登录页 |
| `admin_required` | 管理员权限验证，非管理员显示无权限 |
| `role_required(allowed_roles)` | 角色权限验证，支持多角色配置 |

#### 视图类

| 视图类 | 功能说明 | 权限要求 |
|--------|----------|----------|
| `LoginView` | 用户登录/登出 | 公开 |
| `LogoutView` | 用户登出 | 登录用户 |
| `ChangePasswordView` | 修改密码 | 登录用户 |
| `UserListView` | 用户列表 | 管理员 |
| `UserCreateView` | 创建用户 | 管理员 |
| `UserEditView` | 编辑用户 | 管理员 |
| `UserDeleteView` | 删除用户（禁止删除自己） | 管理员 |

#### 角色体系

| 角色编码 | 角色名称 | 职责描述 |
|----------|----------|----------|
| `admin` | 管理员 | 系统最高权限，所有功能可访问 |
| `business` | 业务人员 | 需求分析、业务管理 |
| `technical` | 技术人员 | 技术实现、测试、元数据维护 |
| `general` | 一般人员 | 只读权限，查看数据 |

### 3.4 AI服务模块 (parameter/ai/ai_service.py)

**文件**: [ai_service.py](file:///d:/CS/trae/PARAM_TOOL/parameter/ai/ai_service.py)

**职责**: AI分析和生成的业务逻辑层

#### AIParameterTableAnalyzer - 参数表分析器

| 方法 | 功能说明 |
|------|----------|
| `analyze_unification(table_id)` | 分析参数表统一到SIMPLELIST表的可行性，输出评分、原因、建议 |
| `analyze_normalization(table_id)` | 检查字段定义规范性，包括命名规范、类型匹配、长度合理性等 |

#### AIGeneratorService - AI生成服务

| 方法 | 功能说明 |
|------|----------|
| `generate_task_document(requirement_id)` | 根据需求生成标准化任务书文档（7个章节） |
| `generate_test_cases(requirement_id)` | 生成测试用例（正常流程/边界条件/异常场景三类） |
| `generate_sql(table_id, natural_query)` | 根据自然语言描述生成SQL语句 |

### 3.5 AI客户端模块 (parameter/ai/ai_client.py)

**文件**: [ai_client.py](file:///d:/CS/trae/PARAM_TOOL/parameter/ai/ai_client.py)

**职责**: AI服务的客户端封装，支持可插拔的AI服务实现

#### BaseAIClient - 基类

| 方法 | 功能说明 |
|------|----------|
| `analyze(prompt, **kwargs)` | AI分析接口（抽象方法） |
| `generate(prompt, **kwargs)` | AI生成接口（抽象方法） |

#### MockAIClient - Mock实现

用于开发和测试环境，模拟AI返回结果，无需实际调用AI接口。

**模拟功能**:
- 参数表统一可行性分析（基于字段数、CODE-NAME模式等规则评分）
- 字段规范性分析（命名、类型匹配、长度等规则检查）
- 任务书生成（标准化模板填充）
- 测试用例生成（按字段生成三类用例）
- SQL生成（根据关键词匹配SQL模板）

## 七、系统特性总结

| 特性 | 说明 |
|------|------|
| 基于角色的访问控制 | 4种角色，细粒度权限控制 |
| 元数据驱动架构 | 通过元数据配置灵活管理参数字段 |

| AI辅助分析 | 参数表统一分析、字段规范性检查 |
| AI智能生成 | 任务书、测试用例、SQL自动生成 |
| Spring Boot代码生成 | 一键生成完整的后端项目代码 |
| 存储方式灵活配置 | code_only / code_name / full 三种模式 |
| 海关参数表示例数据 | 包装种类、运输方式、监管方式等8类参数表 |
