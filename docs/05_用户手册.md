# 参数表配置辅助工具 - 用户手册

## 1. 系统概述

### 1.1 系统简介
参数表配置辅助工具是一款基于Django框架开发的Web应用，旨在帮助金融行业用户高效管理参数表配置，提升参数表开发效率和质量。

### 1.2 主要功能
- 参数表业务说明底账管理
- 元数据配置管理
- 字段定义管理
- 业务需求登记
- 任务书管理
- 配置脚本管理
- INDEXID配置管理
- 测试用例管理
- 自动化测试结果展示
- SQL脚本维护
- AI辅助分析（参数表统一分析、字段规范检查）
- AI智能生成（任务书生成、测试用例生成、SQL生成）

### 1.3 目标用户
- 业务分析师：负责参数表设计和需求分析
- 开发工程师：负责参数表开发和配置
- 测试工程师：负责测试用例设计和执行
- 系统管理员：负责系统维护和管理

## 2. 系统安装与启动

### 2.1 环境要求
- Python 3.9+
- Django 4.2.30
- SQLite 3+

### 2.2 安装步骤

#### 步骤1：安装Python依赖
```bash
pip install django
```

#### 步骤2：执行数据库迁移
```bash
cd PARAM_TOOL
python manage.py migrate
```

#### 步骤3：导入初始数据
```bash
python import_data.py
```

#### 步骤4：启动开发服务器
```bash
python manage.py runserver 8080
```

### 2.3 访问系统
打开浏览器，访问地址：http://127.0.0.1:8080

## 4. 功能使用指南

**使用方法**：

### 4.2 参数表清单

**功能描述**：管理参数表基本信息，支持筛选和搜索

**使用方法**：

#### 查询参数表
1. 点击左侧导航栏"参数表清单"菜单项
2. 使用筛选条件：
   - 业务领域：选择特定领域筛选
   - 状态：选择草稿/启用/废弃状态
   - 搜索框：输入关键词搜索参数表名称或业务说明
3. 点击"搜索"按钮执行筛选

#### 查看参数表详情
1. 在参数表列表中点击目标参数表名称
2. 查看参数表详情：
   - 基本信息（名称、业务说明、领域、负责人等）
   - 字段定义列表（字段名、类型、控件等）
   - 关联INDEXID配置
   - 关联需求列表

#### 参数表状态说明
- **草稿**：参数表正在设计中
- **启用**：参数表已上线使用
- **废弃**：参数表已不再使用

### 4.4 需求登记

**功能描述**：登记和管理参数表相关的业务需求

**使用方法**：

#### 查看需求列表
1. 点击左侧导航栏"需求登记"菜单项
2. 使用状态筛选：待审核/已审核/进行中/已完成/已拒绝
3. 查看需求列表和各状态统计

#### 新建需求
1. 点击页面右上角"新建需求"按钮
2. 填写需求信息：
   - 需求标题：简洁描述需求内容
   - 需求类型：新增/变更/复用
   - 关联参数表：选择对应的参数表（可选）
   - 业务说明：详细描述业务背景和需求
   - 申请人：填写申请人姓名
   - 故事点：估算工作量
   - 冲刺：关联敏捷冲刺（可选）
3. 点击"提交"按钮保存

#### 查看需求详情
1. 在需求列表中点击目标需求标题
2. 查看需求详情：
   - 基本信息
   - 关联参数表信息
   - 关联任务书列表
   - 关联测试用例列表

#### 需求状态流转
```
待审核 → 已审核 → 进行中 → 已完成
              ↘ 已拒绝
```

### 4.10 AI辅助分析

**功能描述**：利用AI技术分析参数表统一可行性和字段定义规范性

**使用方法**：

#### 参数表统一分析
1. 点击左侧导航栏"AI辅助分析"菜单项
2. 在"参数表统一分析"区域：
   - 选择参数表
   - 点击"开始AI分析"按钮
3. 查看分析结果：
   - 统一可行性评分（0-100%）
   - 评分详情（各项得分依据）
   - 建议（是否统一到SIMPLELIST表）
   - 最终推荐

**分析逻辑说明**：
- 字段数量≤8个：加分项
- 包含CODE-NAME字段对：加分项
- 包含VALUE字段：加分项
- 评分≥70%：建议统一到SIMPLELIST表
- 评分50%-70%：建议进一步评估
- 评分<50%：建议定制化开发

#### 字段定义规范性检查
1. 在"字段定义规范性分析"区域：
   - 选择参数表
   - 点击"开始规范检查"按钮
2. 查看检查结果：
   - 问题统计（错误数、警告数）
   - 问题详情列表
   - 建议修改方案

**检查项说明**：
| 检查项 | 说明 | 严重程度 |
|--------|------|----------|
| 命名规范 | 字段名是否使用大写+下划线 | 警告 |
| 类型匹配 | 字段类型与控件类型是否匹配 | 错误 |
| 长度设置 | 字符串长度是否合理 | 警告 |

### 4.11 AI智能生成

**功能描述**：利用AI技术自动生成文档和代码

**使用方法**：

#### 任务书生成
1. 点击左侧导航栏"AI智能生成"菜单项
2. 在"任务书生成"区域：
   - 选择需求
   - 点击"生成任务书"按钮
3. 查看生成结果：
   - 任务书内容（Markdown格式）
   - 包含章节列表
   - 字数统计

**生成内容说明**：
- 需求概述
- 需求信息
- 字段定义表格
- 校验规则
- 技术要求
- 测试要点
- 交付物

#### 测试用例生成
1. 在"测试用例生成"区域：
   - 选择需求
   - 点击"生成测试用例"按钮
2. 查看生成结果：
   - 测试用例统计（总数、正常流程数、边界条件数、异常场景数）
   - 测试用例列表
   - 每个用例包含：用例编号、标题、类型、优先级、前置条件、步骤、预期结果

**生成规则说明**：
- 每个字段生成3条用例：正常录入、边界值测试、为空验证（如必填）
- 用例类型：
  - 正常流程：输入有效值测试
  - 边界条件：输入最大/最小值测试
  - 异常场景：输入无效值或空值测试

#### SQL语句生成
1. 在"SQL语句生成"区域：
   - 选择参数表
   - 输入自然语言查询描述（如：查询所有状态为active的记录）
   - 点击"生成SQL"按钮
2. 查看生成结果：
   - 生成的SQL语句
   - SQL类型（SELECT/INSERT/UPDATE/DELETE）
   - 目标表名

**使用示例**：
| 自然语言描述 | 生成SQL |
|-------------|---------|
| 查询所有记录 | SELECT * FROM parameter_table |
| 统计各状态数量 | SELECT status, COUNT(*) FROM parameter_table GROUP BY status |
| 新增记录 | INSERT INTO parameter_table (...) VALUES (...) |

## 5. 常见问题

### 5.1 无法访问系统

**问题现象**：浏览器访问http://127.0.0.1:8080时无法打开页面

**可能原因**：
- Django开发服务器未启动
- 端口被占用
- 网络连接问题

**解决方法**：
1. 确认开发服务器已启动：查看终端是否显示"Starting development server"
2. 检查端口是否被占用：使用其他端口启动（如python manage.py runserver 8081）
3. 检查网络连接：尝试访问localhost

---

### 5.2 参数表列表为空

**问题现象**：参数表清单页面显示为空

**可能原因**：
- 初始数据未导入
- 数据库迁移未执行

**解决方法**：
1. 执行数据库迁移：`python manage.py migrate`
2. 导入初始数据：`python import_data.py`

---

### 5.3 AI分析无结果

**问题现象**：点击"开始AI分析"后无响应

**可能原因**：
- 未选择参数表
- 网络延迟

**解决方法**：
1. 确保已选择参数表
2. 等待1-3秒，AI分析需要一定处理时间

---

### 5.4 SQL执行失败

**问题现象**：执行SQL语句后显示错误信息

**可能原因**：
- SQL语法错误
- 权限不足
- 数据库锁定

**解决方法**：
1. 检查SQL语句语法
2. 确认操作权限
3. 刷新页面后重新尝试（数据库锁定时）

---

### 5.5 页面样式异常

**问题现象**：页面显示错乱或样式丢失

**可能原因**：
- 浏览器缓存问题
- CSS文件加载失败

**解决方法**：
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 强制刷新页面（Ctrl+F5）
3. 更换浏览器尝试
